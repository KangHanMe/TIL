## 포크한 원격저장소의 브랜치와 원본저장소의 브랜치 병합하는 실습하기

1 ) 개발자 라쿤 깃헙 계정으로 로그인하기  
1 - 1 ) 개발자 라쿤이 커밋한 `like.md`을 확인하기  


2 ) 원본저장소로 풀 리퀘스트 보내기  
2 - 1 ) `[New pull request]`버튼 클릭하기
2 - 2 ) 풀 리퀘스트 보내기 전 정보 확인하기
- `head repository` : 포크한 원본저장소
- `base repository` : 병합할 원격저장소

2 - 2 - 1 ) 풀 리퀘스트 전, 정보를 정확하게 확인하기  
2 - 2 - 2 ) `Create pull request` 버튼 클릭하기  
2 - 2 - 3 ) 풀 리퀘스트에 대한 설명을 적기

3 ) 풀 리퀘스트 완료


## 원본저장소의 소유자가 풀 리퀘스트를 승인하고 병합하는 실습하기

1 ) 원본저장소의 소유자로 깃헙 로그인하기   
1 - 1 ) 원본저장소를 포크한 사람 확인하기  
: 원본저장소의 [Insights] - [Forks]에서 누가 포크했는지 확인 할 수 있다.


1 - 2 ) 원본저장소에서 풀 리퀘스트 확인하기   

- `Files changed` : 풀 리퀘스트에 담긴 새로운 코드를 확인할 수 있다
- `Review changes` : `[write]` - 풀 리퀘스트에 대한 코멘트를 남길 수 있다.
- `Comment` : 댓글만 달기
- `Approve` : 병합하기
- `Request Change` : 수정을 요청하기

1 - 3 ) 풀 리퀘스트 수락하기

- `[Merge pull request]` 버튼 클릭하여 병합 승인하기  
 -성공적으로 병합이 되었다.  
- `[Code]` 탭에서 개발자 너구리의 원격저장소에 푸시한 새롭게 만든 커밋인 `like.md`파일 추가 된 것을 확인할 수 있다.
