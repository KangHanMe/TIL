## 스테이징과 커밋을 수행하는 add, commit

### 기본적인 git 명령들

```sh
git add 파일1 파일2 ...
```

: 파일1 파일2 등, 여러 개의 파일을 스테이지에 추가. 커밋에 파일을 추가한다.

```sh
git commit
```

: 스테이지에 있는 파일들을 커밋한다.

```sh
git commit -a
```

: git add 명령어를 생략하고 바로 커밋한다. 즉, `git add`와 `git commit`을 한 번에 할 수 있다. 단, 변경된 파일과 삭제된 파일은 자동으로 스테이징(스테이지로 올라간다)되고 커밋된다. 파일 상태가 `untracked`일 경우, 커밋이 되지 않는다. (새로 생성한 파일은 스테이지에 올라가지 않는다)

```sh
git push [원격저장소별명] [브랜치이름]
```

: 현재 브랜치에서 새로 생성한 커밋들을 원격저장소에 올린다.

```sh
git pull
```

: 원격저장소의 변경사항을 워킹트리에 반영한다. `git fetch + git merge` 명령이다.

```sh
git fetch [원격저장소별명] [브랜치이름]
```

: 원격저장소의 브랜치와 커밋들을 로컬저장소와 동기화한다. 옵션 생략 시, 모든 원격저장소의 브랜치와 커밋들을 로컬저장소와 동기화한다.

```sh
git merge 브랜치이름
```

: 지정한 브랜치의 커밋들을 현재 브랜치 및 워킹트리에 반영한다.

## 간단한 텍스트 파일 생성하고 확인하기

- CLI 환경인 터미널에서 워킹트리의 경로로 이동한다. `echo "hello git" >`file.txt` 명령어로 "hello git"이라는 텍스트가 담긴 file.txt를 생성한다.
- `ls` 명령어를 통해 현재 워킹트리의 파일 목록을 확인한다.
- `git status` 명령어를 통해 워킹 트리 상태를 확인하다. `file.txt`의 파일 상태는 `untracked`이다.

## 새롭게 생성된 파일을 스테이지에 추가하기

- `git add file.txt` 명령어를 통해 커밋에 `file.txt`파일을 추가한다.
- `git status` 명령어를 통해 `file.txt` 파일은 스테이지 올라간 것을 확인 할 수 있다.

## reset 명령으로 스테이징 취소하기(스테이지에 올리지 않기)

- `git rm --cached <파일이름> ...` 명령어로 파일을 스테이지에 내릴 수 있다.
- 하지만 `git reset [파일명]` 명령어를 사용하면 더 쉽게 파일을 스테이지에 내릴 수 있다.
- 워킹트리의 내용은 변경되지 않는다. 옵션을 생략하면 스테이지의 모든 변경사항을 초기화한다.
- 즉, 워킹트리의 내용을 그대로 두고 해당 파일을 스테이지에 내린다.
- 파일을 스테이지에 내리다 === unstage

## CLI로 커밋 생성하기

- `git add file.txt`
- `git commit` : 커밋 메시지를 작성하는 창의 열린다.
- `git commit -m "커밋에 대한 설명"`
- `git status` 명령어로 워킹트리와 스테이지 영역이 깨끗해진 것을 확인할 수 있다.

## CLI로 커밋 히스토리(이력) 확인하기

- `git log` 명령어로 git의 커밋 이력을 확인할 수 있다.
- `git log --all --graph --decorate ---oneline` 명령을 통해 커밋 아이디 7자리와 커밋 메시지를 한 눈에 확인할 수 있다. 물론 `--all` 옵션을 통해 모든 브랜치의 커밋을 확인할 수 있다.
- 커밋 히스토리에서 커밋 아이디 7자리 숫자는 `커밋 체크섬 또는 커밋 아이디`라고 한다.
- `SHA1 햬시 체크섬`값을 사용한다. 전 세계에 유일한 값이라고 한다. 그리고 커밋 체크섬은 40자리이지만 7자리만 나타낸다.  
  [ 출처 : 도서 팀 개발을 위한 Git, GitHub 시작히기]

## git log의 다양한 옵션 확인하기

- `git log -n<숫자>`
  : 전체 커밋 중에 최신 n개의 커밋 확인하기

- `git log --oneline`
  : 커밋 해시 7자리와 커밋 메시지 확인하기

- `git log --online -n2`
  : 현재 위치한 브랜치의 최신 커밋 2개만 확인 하기

## 도움말 기능 사용하기

- `git help <명령어>`
  : Git의 명령들의 도움말 확인 할 수 있다.
