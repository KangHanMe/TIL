- [v] 변경사항을 잠시 따로 저장해주기

## 변경사항을 잠시 다른 곳에 저장하고, 커밋은 만들지 않는 실습하기

- 예를 들어, 현재 브랜치에서 기능 개발을 했다. 하지만 아직 커밋 하지 않은 파일이 있다. 그리고 커밋은 아직 하고 싶지 않다. 커밋 하지 않은 파일을 잠깐 보관함에 넣어 두고, 다른 작업을 하고 보관함에 넣어뒀던 파일을 다시 꺼낼 수 있게 하는 실습이다.

1 ) 커밋 하지 않은 변경 사항 만들기  
2 ) 커밋 하지 않은 변경 사항을 보관함에 넣어두기

- 소스트리에서 `[스태시]` 클릭 - 보관함에 넣어둘 때의 설명 작성(메시지)
- Git History에서 `stash` 선택 - 보관함에 넣어둘 떄의 설명 작성(메시지)
- `git stash save <보관함 메시지>`작성
- 변경 사항이 잇는 파일들만 stash(넣어두기)된다. 즉, 파일의 상태인 `tracked`만 보관함 넣어진다. 새로 만든 파일은 `untracked` 상태는 보관함에 들어가지 않는다.

## 추가 정보

- `git stash list` : 보관함에 들어간 리스트를 볼 수 있다.
- `git stash drop`: 보관함에 담긴 정보를 삭제할 수 있다.

3 ) 작업이 끝나고 보관함에 있는 정보를 삭제하면, 작업 공간에 변경 사항이 보여진다.
